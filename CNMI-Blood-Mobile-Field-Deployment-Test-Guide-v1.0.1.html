CNMI Blood Mobile — Offline Secure v1.0.1
เป้าหมาย: ยืนยันว่า Notebook เปิดแอป Offline ได้ อ่านบัตร บันทึก คัดกรอง พิมพ์ Barcode และส่งออกข้อมูลได้จริงก่อนเริ่มรับผู้บริจาค

เกณฑ์พร้อมใช้งาน

  • เปิดแอปได้หลังปิด Wi-Fi และ Restart Windows
  • Login และเปิดคลังข้อมูลได้
  • ตั้งชื่องานและช่วงเลขถุงถูกต้อง
  • uTrust 2700R อ่านบัตรได้
  • Import และค้นหาประวัติได้
  • บันทึก คัดกรอง และจ่ายเลขถุงไม่ซ้ำ
  • พิมพ์ A4/Sticker และ Scan Barcode ได้
  • Export LIS CSV และ Backup ได้
  • มี Notebook หรือแบบฟอร์มสำรอง
NO-GO: ห้ามเริ่มด้วยระบบนี้เมื่อ Offline เปิดไม่ได้, Login ไม่ได้, เลขถุงไม่ชัดเจน/เสี่ยงซ้ำ หรือไม่มีแผนเก็บข้อมูลสำรอง

ก่อนออกหน่วยอย่างน้อย 1 วัน

1) เตรียม Browser

  1. ใช้ Windows User และ Browser เดิมที่จะนำไปใช้งานจริง
  2. เปิด https://mobile.cnmiblood.com ขณะออนไลน์
  3. รอหน้าแอปครบ แล้ว Refresh 1 ครั้ง
  4. ห้ามใช้ Incognito/InPrivate และห้ามล้าง Site Data

2) ทดสอบบัญชีและเลขถุง

  1. Logout แล้ว Login ใหม่
  2. เมนู เตรียมออกหน่วย / ส่งออก LIS
  3. กรอกชื่องาน เลขถุงเริ่มต้น–สิ้นสุด และให้เจ้าหน้าที่อีกคนตรวจทาน

3) Import ประวัติ

  1. เลือก Excel/CSV แล้วกดนำเข้า
  2. จดจำนวนรายการสำเร็จ/ข้าม
  3. ค้นหาตัวอย่างอย่างน้อย 3 รายและเทียบข้อมูลต้นทาง

4) ทดสอบ Smart Card

  1. เสียบ uTrust 2700R
  2. ดับเบิลคลิก smartcard-bridge/Start-Bridge.cmd
  3. ต้องเห็น URL http://127.0.0.1:17345
  4. ในแอปกดตรวจสอบเครื่องอ่านบัตร
  5. ทดสอบอ่านบัตรอย่างน้อย 3 ใบก่อนออกหน่วย และสะสม 10 ใบก่อนประกาศใช้จริง

5) ทดสอบข้อมูลจำลอง

  1. ใช้เลขจำลอง เช่น 9999999999991
  2. บันทึกผู้บริจาคจำลอง 2 ราย
  3. ลงผลผ่านและไม่ผ่าน
  4. ตรวจเลขถุงเพิ่มต่อเนื่อง ไม่ซ้ำ และอยู่ในช่วง

6) ทดสอบพิมพ์

  1. พิมพ์ A4 และตรวจข้อมูลไม่ตกขอบ
  2. TSC TE310: 300 dpi, Scale 100%, Margins None, ปิด Fit to page
  3. พิมพ์ Sticker 3 ดวงและ Scan ทุก Barcode

7) ทดสอบ Export

  1. ส่งออก LIS CSV และตรวจจำนวนแถว
  2. ส่งออก Backup เข้ารหัส และตรวจไฟล์ไม่เป็น 0 KB
  3. ทดสอบ LIS Assistant ด้วยข้อมูลจำลองก่อนใช้กับ LIS จริง

ทดสอบ Offline แบบจริง

  1. ปิด Wi-Fi และถอด LAN
  2. ปิด Browser ทุกหน้าต่าง
  3. เปิดใหม่แล้วเข้าแอป
  4. Login บันทึกข้อมูลจำลอง และปิด–เปิดอีกครั้ง
  5. Restart Windows โดยยังไม่เปิด Wi-Fi
  6. เปิดแอป ตรวจข้อมูล เปิด Bridge อ่านบัตร และทดลองพิมพ์
ผ่านเมื่อ: แอปเปิดและบันทึกได้หลัง Restart Windows โดยไม่มีอินเทอร์เน็ต

ตรวจหน้างานภายใน 10 นาที

  • วันที่และเวลา Windows ถูกต้อง
  • Login ได้ และแอปเปิด Offline ได้
  • ชื่องาน/ช่วงเลขถุงถูกต้อง
  • Bridge เปิดและ Reader พร้อม
  • อ่านบัตรทดสอบได้
  • บันทึก/คัดกรองข้อมูลจำลองได้
  • พิมพ์ A4 และ Sticker ได้
  • Scanner อ่าน Barcode ได้
  • Export CSV/Backup ได้
  • ล้างข้อมูลจำลองก่อนเริ่มงานจริง

กรณีต้องเปลี่ยน Notebook

Backup รุ่นนี้ยังไม่มีปุ่ม Restore เข้าเครื่องอื่นโดยตรง จึงต้องเตรียม Notebook สำรองล่วงหน้า
  1. ติดตั้งและทดสอบ Offline บนเครื่องสำรองก่อนเดินทาง
  2. นำเข้าประวัติชุดเดียวกัน
  3. กำหนดช่วงเลขถุงสำรองไม่ให้ทับเครื่องหลัก
  4. ทดสอบ Reader และ Printer
  5. เมื่อเครื่องหลักเสีย ให้หยุดรับรายการ บันทึกเลขสุดท้าย และ Export จากเครื่องหลักหากทำได้
  6. หัวหน้าหน่วยอนุมัติเลขเริ่มต้นของเครื่องสำรอง
  7. ห้ามสองเครื่องจ่ายเลขช่วงเดียวกันพร้อมกัน

หลังจบงาน

  1. กระทบยอดผู้ลงทะเบียน ใบสมัคร และถุง
  2. Export LIS CSV และ Backup เข้ารหัส
  3. นำเข้า LIS และตรวจทานครบก่อน
  4. กดล้างข้อมูลภารกิจ แล้วพิมพ์ CLEAR
  5. ปิด Bridge และบันทึกปัญหา
